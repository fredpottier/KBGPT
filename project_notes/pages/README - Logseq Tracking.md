- ## 📖 Guide d'Utilisation du Tracking Logseq
-
- ### 🎯 Vue d'Ensemble
	- Ce répertoire Logseq track le projet **Back2Promise** : 32 semaines pour revenir à la promesse centrale du système
	- Structure organisée en **7 phases** (0-6) avec dépendances claires
	- Utilise les features natives Logseq : TODO/DOING/DONE, properties, links, tags
-
- ### 📁 Structure des Pages
	- **[[Back2Promise Project]]** - Dashboard principal et vue d'ensemble
	- **[[Phase 0 - Security Hardening]]** à **[[Phase 6 - Production Hardening]]** - Détail des phases
	- **[[ADR Index]]** - Architecture Decision Records
	- **[[Project Blockers]]** - Tracking des blocages
	- **[[BACK2PROMISE_MASTER_ROADMAP]]** - Document de référence complet
	- **[[KG_VALUE_PROPOSITION_CONCRETE]]** - Proposition de valeur du KG
-
- ### 🔄 Workflow Quotidien
	-
	- #### Début de Session
		- 1. Ouvrir journal du jour (ou créer via `Ctrl+Shift+J`)
		- 2. Noter objectifs de la session
		- 3. Identifier quelle phase est en cours
		- 4. Marquer tâches en `DOING`
	-
	- #### Pendant la Session
		- 5. Cocher les tâches complétées (espace sur la checkbox)
		- 6. Ajouter notes techniques dans section appropriée
		- 7. Logger blockers dans [[Project Blockers]] si nécessaire
		- 8. Créer ADR pour décisions importantes
	-
	- #### Fin de Session
		- 9. Résumer réalisations dans journal
		- 10. Marquer tâches `DONE`
		- 11. Noter prochaines étapes
		- 12. Mettre à jour dashboard [[Back2Promise Project]]
-
- ### 🏷️ Tags Utilisés
	- `#p0` - Priorité P0 (bloquant production)
	- `#p1` - Priorité P1 (haute valeur)
	- `#p2` - Priorité P2 (amélioration)
	- `#phase` - Pages de phase
	- `#security` - Tâches sécurité
	- `#adr` - Architecture Decision Records
	- `#blocker` - Blocages actifs
	- `#journal` - Entrées de journal
-
- ### 📊 Mettre à Jour l'Avancement
	-
	- #### Au Niveau Phase
		- Dans la page de phase (ex: [[Phase 0 - Security Hardening]])
		- Modifier `status:: [[PENDING]]` en `status:: [[IN_PROGRESS]]` puis `status:: [[COMPLETED]]`
		- Cocher les critères d'acceptance
	-
	- #### Au Niveau Projet
		- Dans [[Back2Promise Project]]
		- Mettre à jour "Phases Complétées : X/7"
		- Mettre à jour "Semaines Écoulées : X/32"
		- Recalculer "Avancement Estimé : X%"
-
- ### 🔗 Navigation Rapide
	- `[[` pour créer/lier une page
	- `Ctrl+K` pour recherche globale
	- `Ctrl+Shift+G` pour visualiser le graph
	- Clic sur tag (ex: `#p0`) pour filtrer
	- Vue "All pages" pour explorer toutes les pages
-
- ### 📝 Templates Utiles
	-
	- #### Template Nouvelle Tâche
		- ```markdown
		  - TODO [[Titre de la tâche]]
		  	- estimated-effort:: Xh
		  	- assigned-to:: @nom
		  	- due-date:: [[YYYY-MM-DD]]
		  	- description:: ...
		  ```
	-
	- #### Template Blocker
		- ```markdown
		  - BLOCKED [[Description blocker]]
		  	- blocker-id:: B-XXX
		  	- created:: [[YYYY-MM-DD]]
		  	- phase:: [[Phase X]]
		  	- severity:: CRITICAL | HIGH | MEDIUM | LOW
		  	- resolution-plan:: ...
		  ```
	-
	- #### Template ADR
		- ```markdown
		  - #adr #decision
		  - adr-number:: XXX
		  - date:: [[YYYY-MM-DD]]
		  - status:: PROPOSED | ACCEPTED
		  -
		  - ## Contexte
		  - ## Décision
		  - ## Conséquences
		  - ## Alternatives
		  ```
	-
	- #### Template Journal Session
		- ```markdown
		  - ## 📅 Session du [Date]
		  -
		  - ### 🎯 Objectifs
		  - ### ✅ Réalisations
		  - ### 💡 Insights
		  - ### 📝 Notes Techniques
		  - ### 🔜 Prochaine Session
		  - ### 🔗 Références
		  ```
-
- ### 📈 Métriques à Suivre
	- **Velocity** : Tâches complétées par semaine
	- **Blockers actifs** : Nombre et durée moyenne
	- **Phase completion rate** : % avancement de la phase en cours
	- **Drift estimations** : Écart effort estimé vs réel
-
- ### 💡 Tips
	- Faire des commits git réguliers du répertoire `project_notes/`
	- Utiliser Logseq Sync ou Git pour backup
	- Exporter en PDF via `...` → `Export page` si besoin
	- Utiliser queries Logseq pour rapports automatiques
-
- ### 🔍 Queries Utiles
	-
	- #### Toutes les tâches TODO
		- ```clojure
		  {{query (and (todo TODO) (not [[template]]))}}
		  ```
	-
	- #### Blockers actifs
		- ```clojure
		  {{query (and (todo BLOCKED) (page [[Project Blockers]]))}}
		  ```
	-
	- #### Tâches P0
		- ```clojure
		  {{query (and (todo TODO DOING) (page-tags #p0))}}
		  ```
